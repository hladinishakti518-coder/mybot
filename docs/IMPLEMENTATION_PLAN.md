# Пошаговый план реализации MVP

> План для запуска курса «Тазовое дно»: лендинг → бот → оплата → контент → поддержка.

**Реализовано (минимальный MVP):** Этапы 1, 2.3, 3.3, 4, 5.1, 8.1

---

## Текущее состояние

| Компонент | Статус | Что есть |
|-----------|--------|----------|
| Лендинг | ✅ | Next.js, Hero, CourseCards, CTA |
| Бот: старт | ✅ | /start, /activate, /test_pay |
| Бот: оплата | ✅ | /activate по email с проверкой payments, webhook Prodamus + aiohttp |
| Бот: ДЗ | ✅ | /hw, приём видео, пересылка куратору |
| Бот: обратная связь | ✅ | feedback handler |
| Scheduler | ✅ | Проверка доступа, напоминания |
| Google Sheets | ⚠️ | add_user_to_sheet есть, вызов закомментирован |
| Выдача уроков | ✅ | lessons.py — рассылка по schedule.json, дедупликация через lesson_progress |

## Этап 1. Подготовка и конфигурация

### 1.1 Проверить .env

- [ ] `BOT_TOKEN` — токен Telegram-бота
- [ ] `CHANNEL_ID` — ID закрытого канала с уроками
- [ ] `CHAT_ID` — ID чата участников
- [ ] `CURATORS_CHAT_ID` — чат кураторов (для ДЗ)
- [ ] `GOOGLE_CREDENTIALS_FILE` — путь к `google_creds.json`
- [ ] `GOOGLE_SHEET_ID` — ID таблицы
- [ ] `COURSE_DURATION_DAYS` — длительность доступа (например, 21)
- [ ] `NOTIFICATION_DAYS_BEFORE` — за сколько дней напоминать (30, 14, 3)

### 1.2 Исправить баги в payment.py

- [ ] Добавить `from database import get_user` в `approve_request`
- [ ] Проверить: `get_user` возвращает tuple — использовать `user[8]` для `is_active` или переписать на именованный доступ

### 1.3 Ссылка на бота на лендинге

- [ ] В `CtaFixed.jsx` и `Hero.jsx` — кнопка «Начать» ведёт на `https://t.me/BOT_USERNAME`
- [ ] Убедиться, что ссылка ведёт в бота (не в канал)

## Этап 2. Регистрация и онбординг

### 2.1 FSM для сбора данных (опционально)

Сейчас: пользователь вводит `/activate email`. Альтернатива — пошаговый сценарий:

- [ ] /start → «Как тебя зовут?» → «Твой email?» → «Выбери курс» → ссылка на оплату
- [ ] Или оставить текущий flow: оплата на сайте → /activate email

### 2.2 Связка email ↔ tg_id при оплате

- [ ] На странице оплаты (Prodamus/ЮKassa) — поле для ввода `tg_id` или deeplink
- [ ] После оплаты — редирект на `t.me/bot?start=pay_ORDER_ID`
- [ ] Бот по `ORDER_ID` находит email в вебхуке и активирует доступ

### 2.3 Приветственное сообщение после активации

- [ ] После `/activate` или одобрения заявки — отправить инструкцию:
  - как устроен курс
  - когда приходят уроки
  - как сдавать ДЗ
  - ссылка на чат

## Этап 3. Оплата

### 3.1 Выбор платёжной системы

- [ ] Prodamus / ЮKassa / Тинькофф / другая
- [ ] Настроить форму оплаты с полями: email, сумма, курс

### 3.2 Вебхук оплаты

- [ ] Поднять HTTP-сервер (FastAPI/Aiohttp) для приёма POST от платёжки
- [ ] Проверка подписи (HMAC)
- [ ] Сохранение в БД: email, сумма, дата, статус
- [ ] Если передан `tg_id` — сразу вызвать `add_user` и отправить приветствие

### 3.3 Ручная активация (fallback)

- [ ] `/activate email` — поиск оплаты по email в БД
- [ ] Таблица `payments`: email, sum, paid_at, tg_id (заполняется при активации)

## Этап 4. Выдача уроков по расписанию

### 4.1 Загрузка расписания

- [ ] Читать `content_assets/course_tazovoe_dno/schedule.json`
- [ ] Парсить `lessons` с полями: day, lesson_id, title, type, url

### 4.2 Сервис рассылки уроков

- [ ] В `scheduler.py` добавить job: раз в день (например, 09:00 МСК)
- [ ] Для каждого активного пользователя: вычислить `день_курса = (сегодня - payment_date).days`
- [ ] Найти уроки с `day == день_курса`
- [ ] Отправить каждому: текст + ссылку на видео

### 4.3 Формат сообщения урока

- [ ] Заголовок: «Урок N. Название»
- [ ] Описание из schedule.json
- [ ] Кнопка/ссылка на видео

### 4.4 Хранение прогресса (опционально)

- [ ] Таблица `lesson_progress`: user_id, lesson_id, sent_at
- [ ] Не отправлять один и тот же урок дважды
