# Структура проекта: курсы йоготерапевта

> Документ для продюсера курсов. Описание структуры и процессов MVP.

---

## 1. Общая логика

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   Лендинг       │ ──► │   Продажа       │ ──► │   Обучение      │

└─────────────────┘     └─────────────────┘     └─────────────────┘
```

## 3. MVP: что должно работать

| # | Компонент | Назначение |
|---|-----------|------------|
| 1 | Лендинг | Hero, оффер, CTA → переход в бота |
| 2 | Бот: регистрация | Имя, email, выбор курса |
| 3 | Бот: оплата | Ссылка на оплату, проверка статуса |
| 4 | Бот: контент | Выдача уроков по расписанию |
| 5 | Бот: ДЗ | Приём видео/фото, пересылка куратору |
| 6 | Google Sheets | Участники, прогресс, платежи |
| 7 | Обратная связь | Вопросы в боте → куратор |

---

## 4. Данные для хранения

- **Участники**: id, имя, email, курс, дата старта, статус оплаты
- **Прогресс**: пройденные уроки, сданные ДЗ
- **Платежи**: сумма, дата, статус
- **Курсы**: название, длительность, расписание уроков

---

## 5. Контент курса (content_assets)

Каждый курс в `content_assets/course_<slug>/`:

- `schedule.json` — расписание выдачи уроков боту
- `description.md` — описание для лендинга и бота
- `lessons/` — ссылки на видео, PDF, описания
- `homework/` — тексты заданий по урокам
